Cryptology Python Package

Background

One of my favourite films is the Imitation game featuring Benedict Cumberbatch and Keira Knightley. This film tells the story of a mathematician Alan Turing who tries to break the german Enigma Code. The Enigma Code was a method of encrypting messages. This enabled the german army to send messages to each other without fear of messages being intercepted and decoded. I really recommend watching if you haven't had the pleasure.

This lead me to be more interested in the field of cryptology and the history surrounding it. In my Undergraduate degree I chose to study a module in cryptology. The module highlighted the history surrounding cryptology from ancient greece to the present day. There has always been the need for secrecy and I find it curious how these mathematically based ciphers were developed. Below is an ancient technique called Scytale. This focuses on writing a message into a single strip of parchment. The two communicators would use a same size rod and decrypt the message by wrapping the parchment round their rods.

One of my assesments was to decrypt random text using different ciphers. Python and coding in general is a very powerful tool as it allows us to compute and automate processes which can be very time consuming when done manually. I wanted to create an automated process to do this, so started writing some code which allowed me decrypt/encrypt text. The four ciphers I focussed on were atbash, ceaser, playfair and vigenere. Not going to go too much into detail about the specific code but it is all publicaly available here on github. It is split into seperate scripts as below:

Contained within the cryptology.py script are the eight main functions of the ciphers mentioned above (one for encryption and one for decryption). Full docs for these functions can be found on the Readme.MD on the github page. The tools.py script contain mini function which are used within the eight main functions. They break the main functions into smaller, easily managable chunks of code. This makes it easier to debug code as well as unittest code. Every function created within these two files are tested within the tests folder. I used the unittest framework but other frameworks like Pytest are available.

Python Package Structure

The above code helped me answer my assesment questions but one of the great things about Python is the ability to share and distribute code to others. This was my first time attempting this, the guidance I used was a Hitchkiers Guide to packaging code along with this helpful youtube tutorial. Below is the structure of the python package I decided to call cryptology.

Licence

The license tells other developers/users how the package can be used. There are many different types of licences but the one one I decided to use was the MIT license.This allows the code to be distibuted but with a copyright notice.

Mainfest

This ensured that everything from the github repository was packaged up within the package. More on the manifest files can be found on the youtube tutorial mentioned above.

ReadME

This contains information about the package. There is documentation, set up instructions and examples of usage.

Setup.py

This has all the important information needed to set up the package. This needed to be in the directory above the actual code of cryptology I wanted to publish. This includes the version, description, authour details and what the package should ultimately be called.
